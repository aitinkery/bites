# app/ — Capacitor wrap home (not yet active)

This directory is the **structural placeholder** for the future
[Capacitor](https://capacitorjs.com) wrap of Bites.

> **Status:** Not initialized. No `node_modules`, no `ios/` Xcode project,
> no `package.json`. Today, Bites ships as a vanilla PWA at the repo root
> (`/index.html` + `/sw.js` + `/manifest.json`). This folder simply
> reserves the path so the runbook in `../BUILD-IOS.md` has a target.

## Why a placeholder, not a working wrap?

1. **No Apple Developer account yet.** Until that's approved, an Xcode
   project is unbuildable for App Store submission anyway.
2. **The web app is the source of truth.** Every Capacitor build will
   point at `../index.html` as the web asset. No code duplication.
3. **No surprise builds.** If someone runs `npm install` here today,
   they should hit a clear "this isn't initialized yet" surface, not a
   half-broken iOS toolchain.

## What lives here when it's wired up

```
app/
├── README.md              ← this file
├── capacitor.config.json  ← placeholder (real version generated by Capacitor)
├── package.json           ← (TODO: created by `npm init` + `npm install`)
├── ios/                   ← (TODO: created by `npx cap add ios`)
│   └── App/               ← Xcode project
└── node_modules/          ← (gitignored)
```

The web assets (`index.html`, `sw.js`, `manifest.json`, `icons/`,
`vendor/`, `seed-data.json`) **stay at the repo root**. The
`webDir` in `capacitor.config.json` points at `../` so Capacitor
copies the live PWA into the iOS bundle on each build.
